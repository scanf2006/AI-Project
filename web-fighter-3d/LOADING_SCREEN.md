# 加载屏幕功能说明

## 问题
之前用户在加载网页时会短暂看到程序化生成的"积木人"角色，这是因为 3D 模型需要时间加载。

## 解决方案
添加了一个专业的加载屏幕，在模型加载完成前覆盖整个游戏画面。

## 实现细节

### 1. HTML 结构 (index.html)
```html
<div id="loading-screen">
    <div class="loading-content">
        <h1>3D 街头格斗</h1>
        <div class="loading-spinner"></div>
        <p class="loading-text">正在加载角色模型...</p>
        <p class="loading-hint">机器人 vs 特种兵</p>
    </div>
</div>
```

### 2. CSS 样式 (style.css)
- **全屏覆盖**: z-index: 10000
- **渐变背景**: 深蓝色渐变
- **旋转动画**: 霓虹蓝色加载圈
- **渐变文字**: 青色到紫色的标题
- **淡出效果**: 0.5秒平滑过渡

### 3. JavaScript 逻辑

#### Fighter.js
- 模型加载完成后触发 `modelLoaded` 事件
- 每个角色加载完成都会触发一次

#### game.js
- 监听 `modelLoaded` 事件
- 计数器追踪已加载模型数量 (2/2)
- 当所有模型加载完成后:
  1. 等待 0.5 秒
  2. 添加 `hidden` 类（触发淡出动画）
  3. 0.5 秒后移除 DOM 元素

## 用户体验提升

✅ **无闪烁**: 用户不再看到临时的程序化角色
✅ **专业感**: 精美的加载动画
✅ **进度反馈**: 旋转的加载图标
✅ **品牌展示**: 游戏标题和主题展示
✅ **平滑过渡**: 淡入淡出效果

## 技术亮点

- **事件驱动**: 使用 CustomEvent 进行模块间通信
- **异步加载**: 不阻塞游戏初始化
- **性能优化**: 加载完成后移除 DOM 元素
- **CSS 动画**: 使用 GPU 加速的 transform 和 opacity
